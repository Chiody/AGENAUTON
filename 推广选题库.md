## 推广文章选题（按推广节奏排列）

### 第 1 波：认知建立（让人知道这个东西存在）

**选题 1：为什么 90% 的人做 Agent 都在踩坑？**
- 角度：痛点共鸣
- 内容：列举常见翻车场景（幻觉、成本爆炸、记忆丢失），引出"因为没有标准清单"
- 结尾：引导到 Agenauton
- 平台：公众号、知乎、掘金

**选题 2：我用一份清单，3 天做出了一个不翻车的 Agent**
- 角度：实战案例
- 内容：展示自己用 26 维度清单从零构建 Agent 的过程
- 结尾：开源地址
- 平台：B站视频、小红书图文

**选题 3：2026 年了，做 Agent 还在盲目堆 Prompt？**
- 角度：行业观点
- 内容：对比"堆 Prompt"和"标准化构建"的差距
- 结尾：介绍 Agenauton 26 维度
- 平台：Twitter/X、即刻、V2EX

### 第 2 波：深度传播（让人觉得这个东西有用）

**选题 4：拆解 [某知名 AI 产品] 的 26 个维度**
- 角度：拆解学习
- 内容：用 Agenauton 框架分析一个大家都知道的 AI 产品
- 结尾：你也可以用这个框架分析任何产品
- 平台：公众号深度文、知乎专栏
- 注意：可以反复用，每次拆一个不同产品

**选题 5：面试官问"你怎么设计一个 Agent"，我掏出了这张表**
- 角度：职场实用
- 内容：26 维度清单在面试/汇报中的应用
- 结尾：下载清单模板
- 平台：脉脉、知乎、小红书

**选题 6：给 Cursor 一段话，它就能帮你搭好 Agent 架构**
- 角度：AI 编程实战
- 内容：演示把 AI 执行指令丢给 Cursor 的效果
- 结尾：开源地址
- 平台：B站视频、Twitter/X

### 第 3 波：社区裂变（让人参与进来）

**选题 7：全网征集：用 26 维度拆解你最喜欢的 AI 产品**

- 内容：做一个简单的评分表/小程序，用户给自己的 Agent 打分

- 角度：普通人故事
- 内容：记录一个非技术背景的人使用 Agenauton 的完整过程
- 平台：B站、小红书、公众号

## "26 字母 = 26 维度" 营销素材

### 核心 Slogan

> **"26 letters build every word. 26 dimensions build every Agent."**
> **"26 个字母构成所有语言，26 个维度构成所有 Agent。"**

### A-Z 维度对应表（v2.0 彩蛋方向）

如果后续要做"The A-Z of Agentic Intelligence"，可以微调维度英文名使首字母对应 A-Z。当前 26 维度的可能映射方向：

```
A — Architecture Topology（架构拓扑）
B — Boundary of Trust（信任边界）
C — Cost Control（成本管控）
D — Data Schema（数据协议）
E — Evaluation & Testing（评估与测试）
F — Fault Tolerance（容错与恢复）
G — Governance / Quality Loop（质量闭环）
H — Hallucination Suppression（幻觉抑制）
I — Intent Alignment（意图理解）
J — Judgment & Reasoning（决策与规划）
K — Knowledge & Temporal（知识时效管理）
L — Logging & Telemetry（遥测与监控）
M — Multimodal（多模态处理）
N — Nuance / Personalization（个性化适配）
O — Output Experience（输出体验）
P — Planning & Agent Division（Agent 分工）
Q — Query / Instruction Strategy（指令策略）
R — Release & Version（版本与发布）
S — Safety Defense（安全防御）
T — Tool Mapping（工具能力）
U — Upgrade / Self-Evolution（自演化）
V — Verification / Explainability（可解释性）
W — Workflow / Concurrency & Scaling（并发与伸缩）
X — eXplainability（可解释性备选）
Y — Yield / State & Memory（状态与记忆）
Z — Zero-drift / Compliance & Ethics（合规与伦理）
```

注意：这个映射不需要现在定稿，作为 v2.0 的品牌升级方向。强行凑字母会牺牲维度命名准确性，优先保证内容质量。
