## How to Contribute / 贡献方式

### 1. Submit Case Studies 提交案例

最有价值的贡献。使用 Agenauton 标准分析一个 AI Agent，并提交完整的审计报告。

**步骤：**
1. Fork 本仓库
2. 在 `04_Case_Studies/` 下创建案例目录
3. 使用 26 维度测绘表完成分析
4. 提交 Pull Request

### 2. Improve the Framework 完善框架

- 优化 26 维度的定义和描述
- 添加新的探测话术
- 改进术语表
- 修复文档错误

### 3. Build Tools 构建工具

- 自动化探测脚本
- 26 维度在线填报工具
- 架构生成器
- Cursor 规则包

### 4. Translate 翻译

- 将文档翻译为其他语言
- 优化中英文双语表述

### 5. Spread the Word 传播

- 在社交媒体分享 Agenauton
- 撰写博客文章或教程
- 在技术会议上介绍 Agenauton

---

## Pull Request Guidelines / PR 规范

1. **一个 PR 解决一个问题** — 不要在一个 PR 中混合多种改动
2. **清晰的描述** — 说明你改了什么、为什么改
3. **遵循现有格式** — 保持文档风格一致
4. **中英双语** — 核心文档请提供中英文版本

---

## Code of Conduct / 行为准则

- 尊重所有贡献者
- 建设性地讨论分歧
- 遵循伦理准则（参见 [Safety Alignment](./05_Defensive_Lab/safety_alignment.md)）
- 不提交违反法律法规的内容
