## How to Contribute / 贡献方式

### 1. Submit Case Studies 提交案例（最有价值）

用 Agenauton 26 维度标准分析一个 AI Agent，提交完整的审计报告。

**步骤：**
1. Fork 本仓库
2. 在 `04_Case_Studies/` 下创建案例目录
3. 使用 26 维度测绘表完成分析
4. 提交 Pull Request

### 2. Submit Forge Flow Cases 提交铸基六式实战

分享你用「聊找理，投定铸」六步法从想法到产品的完整过程。

**步骤：**
1. Fork 本仓库
2. 参考 `zhujiliushi.html` 中的案例格式
3. 提交你的六步实战记录
4. 提交 Pull Request

### 3. Improve the Framework 完善框架

- 优化 26 维度的定义和描述
- 添加新的探测话术（搜魂/炼魂 9 问）
- 改进术语表
- 修复文档错误
- 优化雷达图维度展示

### 4. Build Tools 构建工具

- 自动化探测脚本
- 26 维度在线填报工具
- 架构生成器
- Cursor / VS Code 规则包

### 5. Translate 翻译

- 将文档翻译为其他语言
- 优化中英文双语表述（所有页面使用 `data-zh` / `data-en` 属性）
- 改进英文内容的地道性（避免直译）

### 6. Improve the Website 改进官网

- `guanwang.html` — 官网首页
- `duoshe.html` — 夺舍大法页面
- `zhujiliushi.html` — 铸基六式页面
- `Agenauton/guide/index.html` — 在线引导
- `lang.js` — 双语切换模块

### 7. Spread the Word 传播

- 在社交媒体分享 Agenauton
- 撰写博客文章或教程
- 在技术会议上介绍 Agenauton

---

## Pull Request Guidelines / PR 规范

1. **一个 PR 解决一个问题** — 不要在一个 PR 中混合多种改动
2. **清晰的描述** — 说明你改了什么、为什么改
3. **遵循现有格式** — 保持文档风格一致
4. **中英双语** — 核心文档请提供中英文版本
5. **测试页面** — 如果修改了 HTML 页面，请在本地打开确认效果
