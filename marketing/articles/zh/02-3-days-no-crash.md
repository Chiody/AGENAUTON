### 我用一份清单，3 天做出了一个不翻车的 Agent

先说结论：**3 天时间，从零到上线，Agent 运行两周零翻车。**

这不是因为我技术多牛，而是因为我在动手之前，花了 2 个小时填了一份表。

#### 背景

我要做的是一个客服 Agent——帮一个电商团队处理售后咨询。需求看起来很简单：用户问问题，Agent 回答。

但如果你做过就知道，"简单"的客服 Agent 是最容易翻车的：
- 用户问退款政策，Agent 编了一个不存在的政策
- 用户发了一张图片，Agent 完全不知道怎么处理
- 两个用户同时问问题，Agent 把上一个用户的订单信息告诉了下一个用户

这些都是真实发生过的事故。

#### 我做了什么不同的事？

在写第一行代码之前，我打开了 Agenauton 的 **26 维度模板**，逐条填写。

**基座层的 4 个决策：**

| 维度 | 我的选择 | 为什么 |
|------|---------|--------|
| 模型选择 | GPT-4o-mini 主力 + GPT-4o 兜底 | 成本控制，复杂问题升级 |
| 知识管理 | RAG + 产品文档向量库 | 确保回答基于真实政策 |
| 记忆系统 | 短期会话记忆 + 用户档案 | 记住当前对话 + 历史订单 |
| 工具集成 | 订单查询 API + 退款 API | 只开放必要的两个工具 |

**认知层的 3 个决策：**

- 推理策略：先检索知识库，再生成回答，最后自检
- 规划能力：简单问题直接答，复杂问题分步处理
- 上下文管理：最近 10 轮对话 + 摘要压缩

**安全层的 3 个决策：**

- 权限控制：Agent 只能查询，不能直接执行退款（需人工确认）
- 输出审查：禁止输出内部系统信息、其他用户数据
- 容错机制：模型调用失败自动重试 3 次，最终转人工

**这些决策花了我 2 个小时。但省了我至少 2 周的返工时间。**

#### 实际构建过程

**Day 1：架构搭建**

把填好的 26 维度模板丢给 Cursor，用 Agenauton 的 `ai_architect_prompt.md` 让 AI 生成了基础架构。Cursor 直接输出了：
- 项目目录结构
- 核心模块划分
- 数据流图
- 关键接口定义

我只需要 review 和微调，不需要从零设计。

**Day 2：核心逻辑**

基于架构骨架，用 Cursor 逐模块实现：
- RAG 检索管道
- 工具调用路由
- 会话管理
- 安全过滤层

因为架构已经想清楚了，每个模块的边界很清晰，写起来很快。

**Day 3：测试上线**

用 Agenauton 的评估维度设计了测试用例：
- 正常问答（准确率）
- 边界情况（图片输入、超长对话）
- 安全测试（诱导泄露、权限越界）
- 并发测试（多用户同时对话）

全部通过后上线。

#### 两周后的数据

- 准确率 94%
- 零安全事故
- 用户满意度 4.6/5
